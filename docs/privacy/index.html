Privacy policy

Last updated July 19, 2026 · Applies to MotoCare 1.0.1

MotoCare is designed as an offline, single-device Android application.

← MotoCare home

Data handling

  • Motorcycle profiles, maintenance history, costs, document details, and preferences are stored locally on the device.
  • MotoCare does not require an account and does not send data to a MotoCare server.
  • MotoCare does not include analytics, advertising SDKs, or user tracking.
  • Photos, videos, receipts, backups, and exports are selected or created through Android's system document picker. The app stores URI references rather than media binaries.
  • Optional notifications are generated locally from saved dates, mileage, and preferences.

User control

You control backup and export destinations. Restoring a JSON backup replaces current records only after explicit confirmation. Uninstalling MotoCare removes its private local database and preferences. Files exported through the document picker remain wherever you saved them.

Permissions

MotoCare requests notification permission on supported Android versions. It does not request network, broad storage, location, contacts, camera, or microphone access.

Android may declare non-runtime WorkManager permissions for scheduled local reminders, including wake lock, boot completion, and foreground-service support.

Changes and contact

This policy will be updated if MotoCare's data handling changes. Privacy or support questions can be sent to motocaresupport@icloud.com.
